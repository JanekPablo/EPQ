27 -  03 november- december 2017

Outcome of Code Test

Part of the process was to analysing this code and understanding what I had to do to improve it and what I needed to research further.
This console code runs in such a way that the user inputs the lux value, the aperture and the films reciprocity factor and outputs a shutter speed. The most complex part of this code is the if statement which ensures the code only takes the reciprocity factor into account if the shutter speed is longer than 1 sec. The code could be easily changed to allow for an output of Aperture with the input of Shutter Speed. However, there are a couple of problems with the code, firstly it is set for an ISO of 100, which should really be a variable dependent on the film so shouldn’t be a fixed value. Another problem is that the output shutter value is a decimal which, is ok if the value is above 1 sec as the value can be rounded and result in an understandable setting for the photographer, however ideally when below 1 sec the aim is that the value defaults to the cameras set shutter intervals which should be given in fractions as it is given on the camera. These two points I will work on and further research is needed in terms of how this can be achieved.

Raspberry Pi Seup

Having received the Raspberry Pi, I started to set this up. This firstly involved formatting a SD card and downloading the Raspian operating system and transfer the system onto the SD Card. Given that the Raspberry Pi Zero was WIFI enabled I could potential wok on it using an SSH connection meaning I wouldn’t have to plug it into my monitor or attach a keyboard. To set this up before it is booted up it needs to automatically connect with our WIFI to allow for this I followed this tutorial, https://www.youtube.com/watch?v=LlCr09B2HZI which shows which files need to be altered or added to allow for this quick wireless setup. However, I found it didn’t work so I tried again by wiping the SD card and re installing the raspian image. But it didn’t work so I resorted to plugging the raspberry pi into my monitor and using a wired keyboard to interact with it. This clearly worked, and I was able to connect it to our WIFI and change the hostname, meaning when I powered it on next time I could SSH into it. Having setup, the Pi the next step was soldering on the pin board to the GPIO (General Pin Input Output) connectors.
Having considered my options when it comes to trying to wirelessly connect the raspberry pi with a light meter application, I thought that a good solution could be to start the process with the Lux Value being displayed on the Pi via screen so the user could manually input the Lux value then later on in the [project I could explore various solutions to the automatic connection between raspberry pi and app. To allow for this I ordered an Adafruit PiOLED - 128x32 Monochrome OLED screen this could directly plugin to these pin connectors soldered onto the pi, and display information. There were a couple of platforms I found that supported the use of connecting the raspberry pi to an application. They are referred to as an IOT (Internet of Things) platform. One of these which was of interest was a piece of software called Cayenne My Devices. Which when downloaded onto the raspberry pi would connect to an online dashboard where you could drag and drop various switches or graphs to control or share data from the pi.
